# OpenTheChests Module

## Goal

This module is the **Member 1 contribution** of the project.

Its purpose is to process `OpenTheChests` observation histories and predict the chest-opening decision required by the environment.

This is the **activity-recognition and decision-making layer** of the project.

---

## Scope

This module covers:

- environment registration for `OpenTheChests-v0`, `OpenTheChests-v1`, and `OpenTheChests-v2`
- oracle-labeled dataset generation
- baseline training and evaluation
- final inference API for the next team members

It does **not** include the KUKA control module.

---

## Main environments

The module supports:

- `OpenTheChests-v0`
- `OpenTheChests-v1`
- `OpenTheChests-v2`

These correspond to increasing difficulty levels.

## Data generation

Oracle labels are generated by evaluating all 8 possible 3-bit actions at a given state and selecting the best action according to immediate reward.

This produces supervised training data of the form:

- **input** = current observation or observation history
- **target** = best action

The final large datasets are:

- `oracle_dataset_v0_large.jsonl`
- `oracle_dataset_v1_large.jsonl`
- `oracle_dataset_v2_large.jsonl`

Train/test splits were performed **by seed**, not by individual sample, in order to avoid leakage between highly similar states from the same episode.

---

## Models explored

Three model families were evaluated:

### Rule-based baseline
A simple hand-crafted mapping from event information to chest-opening action.

### MLP baseline
A feedforward model using the **current observation only**.

### LSTM baseline
A recurrent model using **prefixes of observation histories**.

---

## Final model selection

Current recommended setup:

- `OpenTheChests-v0` -> **MLP**
- `OpenTheChests-v1` -> **MLP**
- `OpenTheChests-v2` -> **LSTM**

This policy is implemented in `predict.py` through `mode="auto"`.
